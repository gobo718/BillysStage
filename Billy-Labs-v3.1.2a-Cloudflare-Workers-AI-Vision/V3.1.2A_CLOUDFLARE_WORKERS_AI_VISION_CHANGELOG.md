# Billy Labs v3.1.2a — Cloudflare Workers AI Vision

## Purpose
Corrects the first vision release so Billy Labs uses the Cloudflare infrastructure already attached to the project rather than requiring a separate OpenAI developer account.

## Changes
- Replaced the OpenAI Responses API adapter with the Cloudflare Workers AI binding.
- Added `[ai] binding = "AI"` to the Worker configuration.
- Defaulted visual analysis to `@cf/meta/llama-3.2-11b-vision-instruct`.
- Preserved permanent field IDs, ranked alternatives, exact confidence percentages, and the 🌈 🟢 🟡 🔴 🪦 visual scale.
- Added defensive image fetching, image-type validation, size limits, JSON extraction, and provider error messages.
- Updated health reporting to identify Cloudflare Workers AI as the configured provider.
- Removed the requirement for an OpenAI API key.

## Deployment note
The existing `CURATOR_PUBLISH_KEY` remains unchanged. Cloudflare may require one-time acceptance of the Meta model license before the first normal analysis request.
